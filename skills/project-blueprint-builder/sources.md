# Sources

This skill is designed as a reusable planning package for software projects.

## Recommended usage

- Use the skill for greenfield product planning.
- Use it for major feature initiatives and technical handoffs.
- Adapt the templates to fit the current codebase and team process.
- Keep the output lean and decision-focused rather than encyclopedic.

## Notes

The skill is intentionally simple, portable, and easy to adapt so it can be reused across teams and published as a standalone skill package.
