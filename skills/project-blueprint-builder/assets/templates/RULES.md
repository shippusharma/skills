# Rules

## 1. Engineering Rules

- Prefer simple, maintainable solutions over over-engineering.
- Keep modules focused on one responsibility.
- Write tests for critical paths and user-facing flows.

## 2. Code Style

- Use clear naming conventions.
- Keep functions small and readable.
- Avoid hidden side effects.

## 3. Testing Expectations

- Unit tests for core logic
- Integration tests for critical workflows
- End-to-end tests for user journeys when appropriate

## 4. Security Rules

- Never expose secrets in code.
- Validate and sanitize user input.
- Apply least-privilege access controls.

## 5. Collaboration Rules

- Document important decisions.
- Keep PRs focused and reviewable.
- Update the memory file when context changes.
