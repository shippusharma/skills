# Blueprint Workflow

## Recommended sequence

1. Start with the product problem and user needs.
2. Define project scope and success metrics.
3. Draft the architecture and system boundaries.
4. Capture implementation rules and engineering standards.
5. Break the work into phases.
6. Document the experience and design direction.
7. Preserve the context in the memory file.

## Quality bar

A good blueprint should be:

- Clear enough for implementation to begin
- Specific enough to avoid ambiguity
- Structured enough for team collaboration
- Flexible enough to evolve as the project matures
